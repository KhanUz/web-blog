
- Node.js 20 及以上版本；
- npm 包管理器；
- 本地 MongoDB 服务。

```bash
npm run dev:frontend
```

## 三、项目概述

本项目实现了一个面向内容发布与阅读场景的博客系统，覆盖文章展示、文章编辑、文章管理、搜索检索、评论互动以及账户管理等核心能力。项目以“内容优先、结构清晰、交互适度”为设计原则，采用服务端渲染为主、前端渐进增强为辅的技术路线，使系统在保证实现复杂度可控的前提下，具备较完整的内容管理能力和较好的阅读体验。

从工程实现角度看，本项目并非传统意义上的前后端完全分离单页应用，而是采用 `Express + TypeScript` 在服务端直接组织页面输出，再通过 `HTMX` 与少量浏览器端脚本对局部交互进行增强。该方案较适合博客这类以内容浏览、表单提交和后台维护为主的系统，能够在开发效率、页面首屏完整性和维护成本之间取得较好平衡。

## 四、项目目标与实现范围

本项目围绕博客系统的典型使用流程进行设计，主要实现内容如下：

- 面向站点所有者的文章管理页与文章编辑页；

- 基于 HTMX 的页面导航增强和局部刷新能力；
- 面向接口调用的 REST 风格 API，用于文章、用户和元信息访问。

## 七、系统总体架构

### 1. 架构模式

本项目采用“服务端渲染 + 渐进增强”的架构模式。其核心特点如下：

- 页面主体由后端直接生成完整 HTML；
- 浏览器首次访问即可获取完整页面结构；
- 前端仅对编辑器预览、目录高亮、导航切换等行为进行增强；
- 局部更新通过 HTMX 完成，而非依赖单页应用状态管理。

### 2. 分层结构

系统可以概括为以下四层：

1. 表现层：由 `ui/pages`、`ui/components` 和 `ui/core` 组成，负责页面结构与展示输出；
2. 路由层：由 `routes/web` 和 `routes/api` 组成，负责请求分发、参数解析和权限校验；
3. 业务支撑层：由 `lib` 下的认证、数据库、序列化、异常处理等模块组成；
4. 数据层：由 `models` 中的 Mongoose 模型与 MongoDB 数据库存储组成。

### 3. 请求处理流程

系统页面请求的一般处理过程如下：

1. 浏览器发起页面或接口请求；
2. `app.ts` 创建的 Express 实例接收请求，并依次执行中间件；
3. `resolveCurrentUser` 中间件解析当前会话，写入 `request.currentUser` 与 `request.actorRole`；
4. 路由模块根据 URL 将请求分发至对应处理函数；
5. 路由函数调用 Mongoose 模型查询或更新数据库；
6. Web 路由调用 `renderXxxPage()` 和 `renderDocument()` 生成完整 HTML；
7. 浏览器接收页面后，由前端 `entry.ts` 对局部交互进行增强。

该流程体现了本项目“后端负责页面组织，前端负责轻量增强”的总体思路。

## 八、后端实现说明

### 1. 服务启动机制

后端启动入口位于 `apps/backend/src/server.ts`。其执行流程为：

1. 调用 `connectToDatabase()` 建立 MongoDB 连接；
2. 调用 `seedDatabase()` 初始化演示数据与缺失数据修复逻辑；
3. 创建 Express 应用实例；
4. 监听配置端口并启动服务。

其中，种子数据模块不仅负责首次插入示例用户、文章、评论和站点资料，也包含一定的数据兼容逻辑，例如对旧角色值进行迁移，以及为缺失密码的账户补全默认密码。这使系统在开发阶段能够快速进入可运行状态。

### 2. 应用装配与中间件

`apps/backend/src/app.ts` 负责装配整个 Express 应用。当前实现中主要包含以下中间件与能力：

- `cors()`：处理跨域访问；
- `express.json()` 与 `express.urlencoded()`：处理 JSON 和表单请求体；
- `morgan("dev")`：输出开发期请求日志；
- `resolveCurrentUser`：解析登录状态与角色信息；
- `/vendor` 静态目录：暴露 `htmx.org` 浏览器端脚本；
- `/assets` 静态目录：暴露前端构建后的样式与脚本资源。

在路由注册顺序上，系统先挂载 `/api/health`、`/api/articles`、`/api/meta`、`/api/users`，再挂载页面型的 `siteRouter`。末尾设置 404 和统一错误处理逻辑，用于返回标准化错误响应。

### 3. Web 路由与页面输出

页面路由集中定义在 `apps/backend/src/routes/web/site.ts`。该文件承担了本系统最核心的页面业务组织职责，主要包括：

- 首页 `/`；
- 最新可访问文章入口 `/article`；
- 文章详情页 `/articles/:slug`；
- 评论提交 `/articles/:slug/comments`；
- 管理页 `/manage`；
- 编辑页 `/editor` 与 `/editor/:id`；
- 搜索页 `/search`；
- 关于页 `/about`；
- 账户页 `/account`；
- 注册、登录、登出与资料更新相关表单提交路由。

`site.ts` 中的 `sendPage()` 函数是页面输出的统一入口。路由函数先查询所需数据，再调用 `renderHomePage()`、`renderArticlePage()`、`renderEditorPage()` 等页面函数生成主体内容，最后通过 `renderDocument()` 包裹统一导航、资源引用和通知提示，形成完整 HTML 文档返回浏览器。

需要说明的是，本项目没有使用 Handlebars、EJS 等模板引擎，而是直接使用 TypeScript 函数返回 HTML 字符串。这种实现方式使页面结构、业务条件和数据绑定逻辑集中在同一语言体系中，便于小型项目维护。

### 4. API 路由设计

除了页面访问，系统还保留了 REST 风格接口，主要位于 `apps/backend/src/routes/api/`：

- `articles.ts`：提供文章列表、按 slug 查询、文章创建与更新、发布、撤回、逻辑删除、评论读取与评论提交；
- `users.ts`：提供会话查询、注册、登录、登出、当前用户资料读取与更新；
- `meta.ts`：提供站点资料等辅助元数据；
- `health.ts`：提供健康检查。

接口层返回的数据通过 `lib/api/serialize.ts` 进行序列化，避免将 Mongoose 文档对象直接暴露给外部调用方。这种做法使接口输出结构更稳定，也更利于后续扩展。

## 九、数据模型设计

系统核心数据模型定义于 `apps/backend/src/models/`，主要包括以下四类实体：

### 1. Article

`ArticleModel` 是博客系统的核心模型，主要字段包括：

- `title`：文章标题；
- `slug`：文章路径标识，用于友好 URL；
- `summary`：文章摘要；
- `content`：Markdown 正文；
- `categories` 与 `tags`：分类和标签；
- `status`：文章状态，支持 `draft`、`published`、`recalled`；
- `publishedAt`：发布时间；
- `removedFromSiteAt`：逻辑删除时间；
- `author` 与 `authorName`：作者关联信息。

其中，`title`、`summary`、`content`、`tags`、`categories` 上建立了文本索引，便于搜索页和接口检索功能实现。`removedFromSiteAt` 的存在说明系统采用逻辑删除，而不是物理删除，这样更有利于管理端保留记录。

### 2. User

`UserModel` 保存账户信息，主要字段包括：

- `email`、`name`；
- `role`：支持 `viewer` 和 `owner`；
- `avatarUrl`、`bio`；
- `passwordHash`：密码哈希值；
- `sessionTokenHash`：会话令牌哈希值；
- `active`：账户是否启用。

该设计体现了较清晰的认证边界：系统不会直接存储明文密码，也不会直接存储原始会话 token，而是存储其哈希值。

### 3. Comment

`CommentModel` 用于保存文章评论，字段相对简洁，包含所属文章、评论者名称、评论内容和状态。当前实现只支持 `approved` 状态，适合作为博客评论功能的基础版本。

### 4. Profile

`ProfileModel` 用于保存博客站点或作者页展示信息，包括姓名、身份描述、个人简介、主题关键词和外部链接。首页与关于页都依赖该模型输出站点介绍内容。

## 十、认证与权限控制实现

### 1. 会话解析

认证逻辑主要位于 `apps/backend/src/lib/auth/`。其中：

- `session.ts` 负责 cookie 解析、Bearer Token 解析、写入与清除会话 cookie；
- `auth.ts` 负责解析当前用户、执行登录态注入和权限校验；
- `passwords.ts` 负责密码哈希、校验与会话 token 生成。

`resolveCurrentUser()` 中间件会优先从 `Authorization` 请求头或 cookie 中读取 token，再查找对应 `sessionTokenHash` 的用户。如果找到用户，则将用户对象挂载到 `request.currentUser`，并将角色写入 `request.actorRole`；否则按访客身份处理。

### 2. 权限模型

本项目的运行时访问角色包括：

- `guest`：未登录访客；
- `viewer`：普通已登录读者；
- `owner`：站点所有者。

其中，真正保存在数据库中的用户角色是 `viewer` 和 `owner`，`guest` 仅作为请求阶段的临时身份标记使用。

### 3. 权限约束逻辑

`auth.ts` 中定义了三类关键权限函数：

- `requireAuthenticated()`：要求必须登录；
- `requireRole()`：要求用户具备指定角色；
- `ensureArticleAccess()` 与 `ensureArticleManagement()`：用于文章访问与文章管理权限控制。

例如，在文章访问场景下，已发布文章对普通读者可见，而草稿、撤回文章或被移出站点的文章，仅允许所有者或作者本人访问；在文章管理场景下，所有者只能管理自己名下的文章。这些规则保证了系统的数据可见性与操作边界。

## 十一、页面与 UI 组织方式

### 1. 统一页面外壳

`apps/backend/src/ui/core/shell.ts` 定义了统一页面外壳。`renderDocument()` 负责输出完整 HTML 文档，`renderAppShell()` 负责导航栏、通知区和主体布局组织。所有页面均复用这一外壳，从而保证：

- 页面结构一致；
- 导航行为一致；
- 通知提示风格一致；
- HTMX 导航增强逻辑可统一挂载。

该外壳在最外层容器上配置了 `hx-boost`、`hx-target`、`hx-select`、`hx-swap` 和 `hx-push-url`，从而使普通链接和表单在兼容服务端渲染的前提下，也能获得更顺滑的局部导航体验。

### 2. 页面生成方式

页面级输出函数位于 `apps/backend/src/ui/pages/`，例如：

- `home.ts`：输出首页；
- `article.ts`：输出文章详情页；
- `manage.ts`：输出管理页；
- `editor.ts`：输出编辑页；
- `search.ts`：输出搜索页；
- `about.ts` 与 `account.ts`：输出关于页和账户页。

这些页面函数不直接处理数据库访问，而是只负责根据已准备好的数据生成 HTML。这样就把“数据准备”和“页面展示”分离开来，使代码职责更清晰。

### 3. 组件复用

可复用组件位于 `apps/backend/src/ui/components/`。例如文章卡片、评论区、目录、表单与通用展示组件都在该目录下集中管理。这种拆分方式有利于：

- 减少重复 HTML 结构；
- 保持不同页面的视觉一致性；
- 降低页面级函数复杂度。

## 十二、Markdown 与目录功能实现

### 1. Markdown 渲染

Markdown 相关逻辑位于 `apps/backend/src/ui/core/markdown.ts`。系统通过 `marked` 将文章正文从 Markdown 转换为 HTML，并在转换过程中自定义了渲染器，实现了以下功能：

- 自动处理标题层级；
- 为标题生成唯一锚点 ID；
- 自动提取 1 至 3 级标题作为目录数据；
- 对链接输出增加安全属性。

因此，文章正文的渲染不是简单文本转 HTML，而是与目录导航数据生成同时完成的。

### 2. 文章目录生成

在 `ui/pages/article.ts` 中，文章详情页调用 `renderArticleContent()` 获取两类结果：

- 渲染后的文章 HTML；
- 对应的目录结构 `toc`。

随后，系统将目录数据交给目录组件渲染，并额外附加 `Comments` 锚点，使评论区能够纳入页面导航结构。该设计对长文阅读场景较为友好。

### 3. 编辑器实时预览

浏览器端的 Markdown 预览逻辑位于 `apps/frontend/src/entry.ts`。该文件会监听带有 `data-editor-shell` 标识的编辑器区域，在用户输入时：

- 读取文本框 Markdown 内容；
- 调用 `marked.parse()` 生成 HTML；
- 将结果写入预览区域；
- 同步调整文本框高度。

因此，编辑页实现了“输入区 + 预览区”的即时联动效果。

## 十三、前端增强逻辑说明

### 1. HTMX 集成方式

本项目没有将 HTMX 作为独立业务框架使用，而是将其作为服务端页面的增强工具。其主要用途包括：

- 导航切换时减少整页跳转割裂感；
- 表单提交后局部替换页面容器；
- 管理操作后即时展示反馈结果；
- 在不引入前端状态管理框架的情况下提升页面流畅度。

### 2. TOC 高亮同步

`apps/frontend/src/entry.ts` 中的 `setupArticleToc()` 会在文章页查找目录面板和锚点链接，根据滚动位置判断当前激活的章节，并为对应目录项写入 `data-active` 状态。这使目录不仅可以跳转，也具备阅读定位提示功能。

### 3. HTMX 交换后的重新初始化

由于页面使用 HTMX 进行局部替换，原本在初次加载时绑定的事件监听器可能失效，因此 `entry.ts` 同时监听：

- `DOMContentLoaded`；
- `htmx:load`；
- `htmx:afterSwap`。

## 十五、典型功能实现分析

### 1. 文章发布与管理流程

所有者进入 `/editor` 后，可以填写标题、摘要、正文、标签和分类等内容。提交表单后，`site.ts` 中的 `/manage/articles` 或 `/manage/articles/:id` 路由会：

1. 校验当前用户是否为 `owner`；
2. 校验标题、摘要、正文等必填字段；
3. 对 slug 进行规范化处理；
4. 将标签与分类字符串转为数组；
5. 根据意图保存为草稿或直接发布；
6. 将保存结果重新渲染回编辑页或管理页。

文章状态设计为 `draft`、`published`、`recalled`，并通过 `removedFromSiteAt` 控制是否从站点中移除，体现了较清晰的内容生命周期管理思路。

### 2. 搜索功能实现

搜索页由 `/search` 路由提供。后端会读取查询参数 `q`，并在文章集合中基于标题、摘要、正文、标签和分类执行正则匹配。同时，系统使用 MongoDB 聚合统计标签频次，为页面提供标签检索入口。这说明搜索模块不仅具备文本检索，也兼顾了主题导航能力。

### 3. 评论提交流程

文章页的评论提交由 `/articles/:slug/comments` 处理。流程如下：

1. 根据文章 slug 查找文章；
2. 校验文章是否可访问；
3. 要求用户角色为 `viewer` 或 `owner`；
4. 创建评论记录；
5. 重新渲染文章页并提示提交成功。

这一实现保持了服务端控制的一致性，同时结合 HTMX 时也能够获得更平滑的交互反馈。

### 4. 账户注册与登录流程

注册和登录流程分别通过 `/auth/register` 与 `/auth/login` 完成。系统会在成功认证后：

- 生成 session token；
- 将 token 进行哈希后保存到数据库；
- 通过 `Set-Cookie` 将会话写入浏览器；
- 更新当前请求上下文中的用户信息；
- 返回新的页面状态或 JSON 会话结果。

这种实现兼顾了页面场景和接口场景，具备较好的统一性。

```ts
import "./styles/global.css";
import { marked } from "marked";

type PreviewScope = ParentNode;
let cleanupToc: (() => void) | null = null;

marked.setOptions({
    gfm: true,
    breaks: true,
});

function syncEditorHeight(textarea: HTMLTextAreaElement): void {
    textarea.style.height = "auto";
    textarea.style.height = `${textarea.scrollHeight}px`;
}

function renderMarkdown(markdown: string): string {
    return marked.parse(markdown) as string;
}

function getEditorShells(scope: PreviewScope): HTMLElement[] {
    const shells = Array.from(
        scope.querySelectorAll<HTMLElement>("[data-editor-shell]"),
    );

    if (scope instanceof HTMLElement && scope.matches("[data-editor-shell]")) {
        return [scope, ...shells];
    }

    return shells;
}

function getTocPanels(scope: PreviewScope): HTMLElement[] {
    const panels = Array.from(
        scope.querySelectorAll<HTMLElement>("[data-toc-panel]"),
    );

    if (scope instanceof HTMLElement && scope.matches("[data-toc-panel]")) {
        return [scope, ...panels];
    }

    return panels;
}

function setupEditorPreview(scope: PreviewScope): void {
    const shells = getEditorShells(scope);

    shells.forEach((shell) => {
        const input = shell.querySelector<HTMLTextAreaElement>(
            "[data-editor-input]",
        );
        const output = shell.querySelector<HTMLElement>(
            "[data-editor-preview]",
        );

        if (!input || !output || shell.dataset.previewReady === "true") {
            return;
        }

        const syncPreview = () => {
            output.innerHTML = renderMarkdown(input.value);
            syncEditorHeight(input);
        };

        syncPreview();
        input.addEventListener("input", syncPreview);
        shell.dataset.previewReady = "true";
    });
}

function setupArticleToc(scope: PreviewScope): void {
    const panels = getTocPanels(scope);

    if (panels.length === 0) {
        cleanupToc?.();
        cleanupToc = null;
        return;
    }

    const panel = panels[0];
    const links = Array.from(
        panel.querySelectorAll<HTMLAnchorElement>(".toc-link[href^='#']"),
    );
    const items = links
        .map((link) => {
            const id = decodeURIComponent(
                link.getAttribute("href")?.slice(1) ?? "",
            );
            const section = id ? document.getElementById(id) : null;

            if (!section) {
                return null;
            }

            return { id, link, section };
        })
        .filter(
            (
                item,
            ): item is {
                id: string;
                link: HTMLAnchorElement;
                section: HTMLElement;
            } => item !== null,
        );

    if (items.length === 0) {
        cleanupToc?.();
        cleanupToc = null;
        return;
    }

    cleanupToc?.();

    const setActive = (activeId: string) => {
        items.forEach(({ id, link }) => {
            link.dataset.active = String(id === activeId);
        });
    };

    const syncActive = () => {
        const threshold = 160;
        let activeId = items[0].id;

        for (const item of items) {
            if (item.section.getBoundingClientRect().top <= threshold) {
                activeId = item.id;
            } else {
                break;
            }
        }

        setActive(activeId);
    };

    links.forEach((link) => {
        link.addEventListener("click", () => {
            const id = decodeURIComponent(
                link.getAttribute("href")?.slice(1) ?? "",
            );

            if (id) {
                setActive(id);
            }
        });
    });

    syncActive();
    window.addEventListener("scroll", syncActive, { passive: true });
    window.addEventListener("resize", syncActive);
    window.addEventListener("hashchange", syncActive);

    cleanupToc = () => {
        window.removeEventListener("scroll", syncActive);
        window.removeEventListener("resize", syncActive);
        window.removeEventListener("hashchange", syncActive);
    };
}

setupEditorPreview(document);
setupArticleToc(document);

document.addEventListener("DOMContentLoaded", () => {
    setupEditorPreview(document);
    setupArticleToc(document);
});

document.body.addEventListener("htmx:load", (event) => {
    const target = event.target;

    if (target instanceof HTMLElement) {
        setupEditorPreview(target);
        setupArticleToc(target);
        return;
    }

    setupEditorPreview(document);
    setupArticleToc(document);
});

document.body.addEventListener("htmx:afterSwap", () => {
    setupEditorPreview(document);
    setupArticleToc(document);
});
```

```ts
export function createApp() {
    const app = express();

    app.use(cors());
    app.use(express.json());
    app.use(express.urlencoded({ extended: true }));
    app.use(morgan("dev"));
    app.use(resolveCurrentUser);
    app.use(

    app.use("/api/meta", metaRouter);
    app.use("/api/users", usersRouter);
    app.use(siteRouter);

    app.use((request, _response, next) => {
        next(
            new HttpError(
                404,
                `Route not found: ${request.method} ${request.originalUrl}`,
            ),
        );
    });

    app.use(
        (
            error: unknown,
            _request: express.Request,
            response: express.Response,
            _next: express.NextFunction,

                });
                return;
```

### 28. `apps/backend/src/lib/auth/auth.ts`

```ts
import mongoose from "mongoose";
import type express from "express";
import { HttpError } from "../http/httpError.js";
import { UserModel } from "../../models/User.js";
import type { ArticleDocument } from "../../models/Article.js";
import type { UserDocument } from "../../models/User.js";
import type { UserRole } from "../../types/blog.js";
import { hashSessionToken } from "./passwords.js";
import {
    clearSessionCookie,
    readBearerToken,
    readSessionCookieToken,
} from "./session.js";

function getHeaderValue(
    value: string | string[] | undefined,
): string | undefined {
    if (typeof value === "string") {
        return value.trim() || undefined;
    }

    if (Array.isArray(value)) {
        return value[0]?.trim() || undefined;
    }

    return undefined;
}

export async function resolveCurrentUser(
    request: express.Request,
    response: express.Response,
    next: express.NextFunction,
): Promise<void> {
    const authHeader = getHeaderValue(request.header("authorization"));
    const bearerToken =
        readBearerToken(request) ??
        (authHeader?.startsWith("Bearer ")
            ? authHeader.slice(7).trim()
            : undefined);
    const cookieToken = readSessionCookieToken(request);
    const token = bearerToken ?? cookieToken;

    request.currentUser = null;
    request.actorRole = "guest";

    if (!token) {
        next();
        return;
    }

    const sessionTokenHash = hashSessionToken(token);
    const user = await UserModel.findOne({
        sessionTokenHash,
        active: true,
    });

    if (!user) {
        if (bearerToken) {
            next(
                new HttpError(
                    401,
                    "Authenticated session was not found or is inactive.",
                ),
            );
            return;
        }

        if (cookieToken) {
            clearSessionCookie(response);
        }

        next();
        return;
    }

    request.currentUser = user;
    request.actorRole = user.role;
    next();
}

export function requireAuthenticated(request: express.Request): UserDocument {
    if (!request.currentUser) {
        throw new HttpError(401, "Authentication is required for this action.");
    }

    return request.currentUser;
}

export function requireRole(
    request: express.Request,
    roles: UserRole[],
): UserDocument {
    const user = requireAuthenticated(request);

    if (!roles.includes(user.role)) {
        throw new HttpError(
            403,
            "You do not have permission to perform this action.",
        );
    }

    return user;
}

export function ensureArticleAccess(
    request: express.Request,
    article: ArticleDocument,
): void {
    if (article.removedFromSiteAt && request.actorRole !== "owner") {
        throw new HttpError(404, "Article not found.");
    }

    if (request.actorRole === "owner") {
        return;
    }

    if (
        request.currentUser &&
        article.author &&
        article.author.equals(request.currentUser._id)
    ) {
        return;
    }

    if (article.status !== "published" || article.removedFromSiteAt !== null) {
        throw new HttpError(404, "Article not found.");
    }
}

export function ensureArticleManagement(
    request: express.Request,
    article: ArticleDocument,
): UserDocument {
    const user = requireRole(request, ["owner"]);

    if (!article.author || !article.author.equals(user._id)) {
        throw new HttpError(403, "Owners can only manage their own articles.");
    }

    return user;
}
```

### 29. `apps/backend/src/lib/auth/passwords.ts`

```ts
import {
    randomBytes,
    scryptSync,
    timingSafeEqual,
    createHash,
} from "node:crypto";

const SCRYPT_KEY_LENGTH = 64;

export function hashPassword(password: string): string {
    const salt = randomBytes(16).toString("hex");
    const derivedKey = scryptSync(password, salt, SCRYPT_KEY_LENGTH).toString(
        "hex",
    );
    return `${salt}:${derivedKey}`;
}

export function verifyPassword(password: string, storedHash: string): boolean {
    const [salt, expectedHex] = storedHash.split(":");

    if (!salt || !expectedHex) {
        return false;
    }

    const actual = scryptSync(password, salt, SCRYPT_KEY_LENGTH);
    const expected = Buffer.from(expectedHex, "hex");

    if (actual.length !== expected.length) {
        return false;
    }

    return timingSafeEqual(actual, expected);
}

export function createSessionToken(): string {
    return randomBytes(32).toString("hex");
}

export function hashSessionToken(token: string): string {
    return createHash("sha256").update(token).digest("hex");
}
```

```ts
export const SESSION_COOKIE_NAME = "web_blog_session";

function parseCookieHeader(header: string | undefined): Record<string, string> {
    if (!header) {
        return {};
    }

    return header
        .split(";")
        .map((part) => part.trim())
        .filter(Boolean)
        .reduce<Record<string, string>>((cookies, part) => {
            const separatorIndex = part.indexOf("=");
            if (separatorIndex === -1) {
                return cookies;
            }

            const name = part.slice(0, separatorIndex).trim();
            const value = part.slice(separatorIndex + 1).trim();

            if (!name) {
                return cookies;
            }

            cookies[name] = decodeURIComponent(value);
            return cookies;
        }, {});
}

export function readBearerToken(request: express.Request): string | undefined {
    const authHeader = request.header("authorization");
    return typeof authHeader === "string" && authHeader.startsWith("Bearer ")
        ? authHeader.slice(7).trim()
        : undefined;
}

export function readSessionCookieToken(
    request: express.Request,
): string | undefined {
    const cookies = parseCookieHeader(request.header("cookie"));
    return cookies[SESSION_COOKIE_NAME];
}

export function readSessionToken(request: express.Request): string | undefined {
    return readBearerToken(request) ?? readSessionCookieToken(request);
}

function buildCookieValue(value: string, maxAgeSeconds?: number): string {
    const parts = [
        `${SESSION_COOKIE_NAME}=${encodeURIComponent(value)}`,
        "Path=/",
        "HttpOnly",
        "SameSite=Lax",
    ];

    if (process.env.NODE_ENV === "production") {
        parts.push("Secure");
    }

    if (typeof maxAgeSeconds === "number") {
        parts.push(`Max-Age=${maxAgeSeconds}`);
    }

    return parts.join("; ");
}

export function setSessionCookie(
    response: express.Response,
    token: string,
): void {
    response.append("Set-Cookie", buildCookieValue(token, 60 * 60 * 24 * 30));
}

export function clearSessionCookie(response: express.Response): void {
    response.append("Set-Cookie", buildCookieValue("", 0));
}
```

```ts
    request: Request,
    response: Response,
```
```ts
    notice?: Notice,

        typeof request.query.q === "string" ? request.query.q.trim() : "";

        ArticleModel.find({

                          { summary: new RegExp(query, "i") },
                          { content: new RegExp(query, "i") },
                          { tags: new RegExp(query, "i") },
```

```ts
import { resolve } from "node:path";
import { fileURLToPath } from "node:url";

type ViteManifest = Record<string, { file: string; css?: string[] }>;

const currentDir = fileURLToPath(new URL(".", import.meta.url));
const frontendDistDir = resolve(currentDir, "../../../../frontend/dist");
const frontendManifestPath = resolve(frontendDistDir, ".vite/manifest.json");
const frontendDevOrigin =
    process.env.FRONTEND_DEV_ORIGIN ?? "http://127.0.0.1:5173";

let cachedManifest: ViteManifest | null = null;

function readManifest(): ViteManifest | null {
    if (cachedManifest) {
        return cachedManifest;
    }

    if (!existsSync(frontendManifestPath)) {
        return null;
    }

    cachedManifest = JSON.parse(
        readFileSync(frontendManifestPath, "utf8"),
    ) as ViteManifest;
    return cachedManifest;
}

export function getFrontendDistDir(): string {
    return frontendDistDir;
}

export function renderFrontendAssetTags(): string {
    if (process.env.NODE_ENV !== "production") {
        return [
            `<link rel="stylesheet" href="${frontendDevOrigin}/src/styles/global.css" />`,
            `<script type="module" src="${frontendDevOrigin}/@vite/client"></script>`,
            `<script type="module" src="${frontendDevOrigin}/src/entry.ts"></script>`,
        ].join("");
    }

    const manifest = readManifest();
    const entry = manifest?.["src/entry.ts"] ?? manifest?.["index.html"];

    if (!entry) {
        return "";
    }

    const cssTags = (entry.css ?? [])
        .map((href) => `<link rel="stylesheet" href="/${href}" />`)
        .join("");

    return `${cssTags}<script type="module" src="/${entry.file}"></script>`;
}
```

```ts
import { marked } from "marked";

import { escapeHtml } from "./shell.js";

export type TocItem = {
    id: string;
    label: string;
    level: 1 | 2 | 3;
};

function slugifyHeading(value: string): string {
    return value
        .toLowerCase()
        .trim()
        .replace(/[^a-z0-9\s-]/g, "")
        .replace(/\s+/g, "-")
        .replace(/-+/g, "-");
}

function createMarkedRenderer(toc?: TocItem[]) {
    const usedIds = new Map<string, number>();
    const renderer = new marked.Renderer();

    const getUniqueId = (label: string) => {
        const base = slugifyHeading(label) || "section";
        const count = usedIds.get(base) ?? 0;
        usedIds.set(base, count + 1);
        return count === 0 ? base : `${base}-${count + 1}`;
    };

    renderer.link = ({ href, title, tokens }) => {
        const text = renderer.parser.parseInline(tokens);
        const titleAttr = title ? ` title="${escapeHtml(title)}"` : "";
        return `<a href="${escapeHtml(href)}" target="_blank" rel="noreferrer"${titleAttr}>${text}</a>`;
    };

    renderer.heading = ({ tokens, depth }) => {
        const text = renderer.parser.parseInline(tokens);
        const label =
            tokens
                .map((token) => ("text" in token ? String(token.text) : ""))
                .join("")
                .trim() || "Section";
        const level = Math.min(depth, 3) as 1 | 2 | 3;
        const id = getUniqueId(label);

        if (toc) {
            toc.push({ id, label, level });
        }

        return `<h${level} id="${id}">${text}</h${level}>`;
    };

    return renderer;
}

export function renderArticleContent(markdown: string): {
    html: string;
    toc: TocItem[];
} {
    const toc: TocItem[] = [];

    const html = marked.parse(markdown, {
        gfm: true,
        breaks: false,
        renderer: createMarkedRenderer(toc),
    }) as string;

    return { html, toc };
}
```

```ts
            <a class="nav-chip" href="/articles/${escapeHtml(article.slug)}">Open article</a>
          </div>
        </article>
      `,
              )
              .join("")
        : `<p class="text-sm leading-7 text-stone-600">No articles matched the current query.</p>`;
}

        </form>
      </section>

      <div class="grid gap-4">
        ${renderSearchResults(results)}
      </div>
    </section>
  `;
}
```
